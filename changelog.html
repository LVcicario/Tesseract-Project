HISTORIQUE // V1 → V3.0

Ce qui a changé,
et pourquoi.

Le projet est né d'une conversation avec Claude Artifacts. Depuis, il a bougé vite. Cette page trace les étapes. Chaque version répond à une question que le projet s'est posée en chemin.

v3.0 ACADÉMIE 2026-04

La question : comment faire de ce site un outil pédagogique pour les 15–25 ans curieux de blockchain ?

  • Deuxième porte d'entrée : academy.html + 6 chapitres complets (hash, bloc, PoW, identités, consensus, smart contracts)
  • Chaque chapitre suit un modèle pédagogique en 5 temps (Raconter / Montrer / Faire / Défier / Retenir)
  • Défis validés automatiquement, quizz 3 questions avec feedback, progression persistée en localStorage
  • Chapitre 4 : vraie cryptographie ECDSA P-256 via SubtleCrypto, clés non-extractable
  • Chapitre 5 : simulation consensus à 5 nœuds, byzantins togglables, quorum 3/5 · 4/5 · 5/5
  • Chapitre 6 : mini-VM smart contract avec gas visible, limite économique palpable
  • Glossaire flottant accessible sur toutes les pages (15 entrées)
  • Onboarding premier-run, tamper pré-explication, difficulté mining réglable, hash sandbox avec effet avalanche
  • Pages institutionnelles : à propos, docs, changelog (vous êtes ici)
  • Infra : GitHub Actions deploy Pages, CONTRIBUTING/CoC, issue/PR templates, sitemap, robots, OG image SVG, JSON-LD Course

v2.3 HARDENING 2026-04

La question : comment éviter que ce beau site ait des fuites mémoire, des couplages fragiles, des trous d'accessibilité ?

  • Lifecycle registry qui track tous les setInterval et requestAnimationFrame, cleanup sur pagehide (fixe zombie-audio)
  • Event bus explicite entre core et extensions via CustomEvent — remplace trois MutationObserver fragiles
  • Toast + modal primitives (window.TSCUi) — alert() et confirm() remplacés partout
  • safeSetItem wrapper pour localStorage — quota-exceeded visible via toast
  • A11y baseline : aria-label sur CLI, focus trap sur explorer, aria-live sur compteur, wallet accessible clavier
  • Mining non-bloquant : boucle time-budgeted 6 ms + scheduler.yield() / MessageChannel fallback
  • Fix audio toggle : boot-promise guard + labels ON/OFF explicites
  • Fix sélection texte pendant drag tesseract
  • "Sommets illuminés" devient interactif (modal récap mining)

v2.2 REFACTOR 2026-04

La question : comment présenter ce projet sur GitHub avec la même rigueur qu'un projet pro ?

  • Split du monolithe 2000 lignes en index.html + styles/main.css + src/core.js + src/extensions.js
  • Extraction par sed (pas de retranscription manuelle — zéro risque de drift)
  • README complet (244 lignes), LICENSE MIT, .gitignore, favicon SVG standalone
  • Archives des versions monolithiques dans archive/ pour traçabilité

v2.1 BLOCKCHAIN VIVANTE 2026-04

La question : comment transformer une démo visuelle en vraie simulation interactive ?

  • Wallet persistant : identité générée par SHA-256, affichée dans le HUD, survit aux rechargements
  • Mining crédite le wallet +6,25 TSC par bloc, avec sommets illuminés sur le tesseract (jusqu'à 16)
  • Mempool live + sparkline TPS dans la section simulation
  • Explorateur de blocs slide-in avec merkle root visualisé en arbre binaire SVG
  • Mini-VM pour smart contracts (counter, vote) pilotable via CLI
  • Mode x-ray (Konami) : dump JSON de l'état en direct
  • Glitch global + chromatic aberration sur cassure tamper
  • Tesseract section-aware : hue-rotate live selon la section visible

v2.0 PÉDAGOGIE 2026-04

La question : comment rendre ce beau site utile pour apprendre ?

  • Meta SEO complètes, favicon SVG, prefers-reduced-motion, fallback hover: none
  • Enrichissements pédagogiques par section (bandeau live hero, diagramme 1D→4D, chiffres sourcés, 4ᵉ nœud chronologique projectif)
  • Leaderboard local des blocs minés
  • CTAs outro (re-voir, whitepaper, partage X)

v1 ARTIFACT 2026-04

La question : à quoi ressemblerait une présentation de la Blockchain 3.0 comme œuvre d'art ?

  • Monolithe HTML de 1268 lignes généré en une conversation avec Claude Artifacts
  • Tesseract 4D en WebGL (Three.js r128)
  • Minage SHA-256 réel via Web Crypto API
  • Cascade d'invalidation visible à l'œil nu dans la section tamper
  • Audio génératif (Tone.js) + HUD + CLI + boot sequence + curseur custom
