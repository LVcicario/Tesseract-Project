CHAPITRE 06 · ★★★ AVANCÉ

Les programmes.
Du code qui vit.

Jusqu'ici, la blockchain servait à échanger des jetons. Vitalik Buterin a lâché une idée en 2013 : et si la blockchain exécutait aussi du code ? Pas du code n'importe comment : du code déterministe, exécuté par tous les nœuds en même temps, et qui paie pour chaque opération. C'est le smart contract — et c'est ce qui a ouvert la porte à la finance décentralisée, aux NFT, au vote on-chain, et à tout ce qui viendra ensuite.

Durée ≈ 18 minNiveau ★★★ AvancéPrérequis Chapitres 1 à 5
01
Raconter

Un ordinateur partagé par tout le monde.

Un smart contract, c'est un bout de code. Comme n'importe quel programme : des variables, des fonctions, de la logique. Ce qui change, c'est où il tourne :

  • Il est stocké dans un bloc (comme toute transaction).
  • Il est exécuté par chaque nœud du réseau, pas sur un serveur.
  • Son état (ses variables) est écrit dans la chaîne, et tout le monde peut le lire.
  • Chaque opération coûte du gas — payé en monnaie du réseau, prélevé sur l'utilisateur qui déclenche l'appel.

Conséquences fortes : un smart contract ne peut pas mentir sur son état (tout le monde le recalcule), ne peut pas boucler à l'infini (le gas s'épuise), et ne peut pas dépendre de l'extérieur (pas d'appel réseau, pas de date système). Il doit être déterministe : même entrée, même sortie, toujours.

Analogie. Un distributeur automatique. Il a une logique fixe : tu mets la somme, tu tapes le code, tu obtiens le produit. Personne ne peut demander au distributeur de « faire une exception » — il obéit à sa logique, un point c'est tout. Un smart contract, c'est un distributeur mondial, visible par tous, dont personne ne détient la clé. Une fois déployé, il est au-dessus de son créateur lui-même.
02
Montrer

Un compteur qui vit sur la chaîne.

Ci-dessous, tu vas déployer un micro-contrat Counter. Il stocke un entier, et expose trois actions : increment, decrement, reset. Chaque appel consomme du gas (une sorte de carburant payé par l'appelant). Si le gas disponible tombe en-dessous du coût d'une opération, elle est rejetée.

C'est une simplification pédagogique : Ethereum utilise une VM beaucoup plus complexe (l'EVM), avec des milliers d'opcodes et un modèle économique sophistiqué. Le principe, lui, est exactement le même.

▸ LABORATOIRE SMART CONTRACT · COUNTER

Code du contrat

contract Counter { int count = 0; fn increment() { // gas: 5 count += 1; } fn decrement() { // gas: 5 if (count > 0) count -= 1; } fn reset() { // gas: 20 count = 0; } view fn get() { // gas: 0 (lecture) return count; } }

03
Faire · Défier

Épuise le gas.

Ton défi : déployer le contrat, puis appeler increment() suffisamment de fois pour que l'état count atteigne 10 ou plus. Cela va nécessiter 10 × 5 = 50 gas, sur un budget de 100 — donc largement possible sans épuiser. Mais gratte plus bas : essaie de faire déborder le gas en déclenchant reset après plusieurs increment. Observe le rejet.

◆ DÉFI // ATTEINDRE count ≥ 10 EN ATTENTE

Déploie le contrat, puis clique increment() au moins 10 fois. Quand l'état count affiche 10 ou plus, le défi est validé. Tu peux aussi tester les autres opérations pour voir le gas fondre.

05
Retenir

Ce qu'il faut garder.

◆ À RETENIR
  • Un smart contract est un programme stocké et exécuté par la blockchain elle-même.
  • Il est déterministe — même entrée, même sortie, toujours — et ne peut pas dépendre de l'extérieur (pas d'API, pas d'horloge).
  • Chaque opération consomme du gas, payé par l'appelant. Cela empêche les boucles infinies et les attaques par spam.
  • Une fois déployé, un contrat est immuable — même son auteur ne peut plus le modifier, sauf s'il a prévu un mécanisme explicite de mise à jour.
  • Les smart contracts ont donné naissance à la DeFi (finance décentralisée), aux NFT, aux DAO, au vote on-chain. Ethereum en 2015 a été le premier écosystème à les populariser.
◆ QUIZ // 3 QUESTIONS
QUESTION 1 / 3
Pourquoi un smart contract doit-il être déterministe ?
Consensus contraint. Si le contrat faisait appel à Math.random() ou à l'heure système, chaque nœud obtiendrait un résultat différent et le consensus serait impossible.
QUESTION 2 / 3
À quoi sert le gas ?
Double rôle. Incitatif économique (les nœuds sont payés pour exécuter) et mécanisme d'arrêt (une boucle sans fin ruinerait l'appelant avant d'atteindre la fin, donc elle s'arrête).
QUESTION 3 / 3
Quelle blockchain a popularisé les smart contracts ?
Le saut. Bitcoin avait des scripts très limités (non-Turing-complets), utilisés surtout pour signer des transactions. Ethereum a ouvert la Turing-complétude via l'EVM et déclenché l'explosion DeFi / NFT / DAO qu'on connaît.
